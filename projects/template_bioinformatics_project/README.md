## What is this?

A **thin orchestration** bioinformatics project template.

The numbered scripts in `scripts/` are lightweight wrappers that load configuration, call `metainformant.*` library functions, and write structured logs. **No algorithms live in the scripts.** All heavy computation belongs in versioned, testable library modules.

**Key properties:**

- 🔒 **Immutable raw data** — `data/raw/` is never modified by any script
- ♻️ **Idempotent stages** — safe to re-run; existing outputs are preserved unless `--force`
- 🔬 **Zero-Mock testing** — tests run real code on synthetic data, no patching
- 📄 **Config-driven** — all paths and thresholds in `config/default.yaml`; no hardcoding
- 📦 **uv-managed** — reproducible environment via `pyproject.toml` + lockfile

## Quick Start

```bash
# 1. Install uv (if not installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. Set up environment
cd projects/template_bioinformatics_project
uv sync

# 3. Generate synthetic test data
uv run scripts/99_create_synthetic_data.py

# 4. Run the full pipeline
./run.sh
# or equivalently:
uv run main.py --stage all

# 5. Run tests
uv run pytest tests/ -v
```

## Pipeline

### Running Stages

```bash
./run.sh                           # Run all stages
./run.sh --stage process           # Stage 1 only
./run.sh --stage analyze           # Stage 2 only
./run.sh --stage visualize         # Stage 3 only
./run.sh --config config/custom.yaml  # Custom config
./run.sh --synthetic               # Generate synthetic data, then run all
./run.sh --clean                   # Remove all generated outputs
```

### Pipeline Flow

```text
uv run scripts/99_create_synthetic_data.py   → data/raw/samples_*.csv

==> Stage 1: Data Processing (01)
    input:   data/raw/*.csv
    actions: concat → missing-value filter → z-score normalise
    output:  data/processed/processed_data.csv
    log:     logs/01_process_data.log

==> Stage 2: Results Analysis (02)
    input:   data/processed/processed_data.csv
    actions: summary stats → correlation matrix → optional PCA
    output:  results/tables/summary_statistics.csv
             results/tables/correlation_matrix.csv
             results/tables/analysis_metadata.json
    log:     logs/02_analyze_results.log

==> Stage 3: Visualisation (03)
    input:   data/processed/processed_data.csv
             results/tables/correlation_matrix.csv
    actions: distribution grid → correlation heatmap
    output:  results/figures/distribution_grid.png
             results/figures/correlation_heatmap.png
    log:     logs/03_visualize.log
```

Each stage is **idempotent** — add `--force` to any script to unconditionally reprocess.

## Configuration

All parameters live in `config/default.yaml`. Key sections:

| Section | Controls |
| :--- | :--- |
| `metadata` | Project name, organism, version (for provenance logs) |
| `paths` | All input and output directories |
| `processing` | Filtering thresholds, normalisation toggle, thread count |
| `analysis` | Method (`summary` / `correlation` / `pca`), n_components |
| `visualization` | DPI, palette, figure dimensions, file format |
| `logging` | Log level and format |

Create a custom config to override any key:

```bash
cp config/default.yaml config/my_experiment.yaml
# Edit my_experiment.yaml
./run.sh --config config/my_experiment.yaml
```

See [Configuration Reference →](doc/configuration.md)

## Forking This Template

1. Copy this directory and give it a meaningful name.
2. Update `pyproject.toml`: `name`, `description`, `version`.
3. Fill in `config/default.yaml`: organism, genome accession, thresholds.
4. Fill in `config/samples.tsv` with your real sample manifest.
5. Replace synthetic scripts with real data acquisition (see [Data Management →](doc/data.md)).
6. Rename / add numbered scripts for your analysis stages.
7. Update `README.md`, `AGENTS.md`, `SPEC.md`, `doc/`.
8. Add domain-specific `metainformant.*` module calls to your scripts.

## Testing

```bash
# Install dev dependencies
uv sync --extra dev

# Run the full Zero-Mock test suite
uv run pytest tests/ -v

# Run with coverage
uv run pytest tests/ -v --cov=scripts --cov-report=term-missing
```

Tests cover:

- Stage 99 synthetic data generation (files created, valid YAML provenance)
- Stage 01 data processing (output exists, idempotency, handles empty dir)
- Stage 02 analysis (summary CSV, JSON metadata, exits non-zero without input)
- Stage 03 visualisation (PNG files produced)
- Full end-to-end pipeline from synthetic data → final figures

## Principles of Reproducibility

1. **No Hardcoding** — all paths and thresholds route through `config/default.yaml`.
2. **Immutability** — `data/raw/` contents are never modified by any script.
3. **Traceability** — structured logs in `logs/` record inputs, outputs, and timing per run.
4. **Lockfiles** — `uv.lock` pins exact dependency versions for reproducible environments.
5. **Provenance** — `data/raw/metadata.yaml` documents data origin and acquisition date.
6. **Version control** — tag releases with `git tag v1.0.0` and record commit hashes in reports.

## Troubleshooting

| Symptom | Cause | Fix |
| :--- | :--- | :--- |
| `No CSV files found in data/raw/` | Raw data missing | Run `uv run scripts/99_create_synthetic_data.py` |
| Stage 2 exits non-zero | Stage 1 not completed | Run `01_process_data.py` first |
| Stage 3 heatmap missing | Stage 2 not run with `method: summary` | Run `02_analyze_results.py --force` |
| `uv: command not found` | uv not installed | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| Output files in project root | Stray output from old code | Check `paths.data_processed` / `paths.results` in config |
| Tests fail with import errors | Dev deps not installed | `uv sync --extra dev` |
